# Electron RAG 桌面架构

## 设计目标

桌面端知识库应用需要把本地权限、实时交互和 AI 流程拆分清楚。Electron 适合承接窗口、文件系统和本地进程管理；React 负责交互；Python 服务负责检索与记忆。

## IPC 边界

主进程只负责本地系统能力，不直接承载复杂 RAG 逻辑。渲染层通过 preload 暴露的白名单接口访问会话列表、运行时配置和持久化能力。

## 三栏式布局

- 左栏管理 Session 与历史摘要
- 中栏承接聊天流式输出与 Markdown 内容
- 右栏展示当前会话的摘要、长期记忆命中与 RAG 来源

## 流式体验

桌面应用要尽量把等待过程显式化。SSE 或 WebSocket 都可以，当前实现使用 SSE，把后端增量回答逐段写入聊天气泡。
