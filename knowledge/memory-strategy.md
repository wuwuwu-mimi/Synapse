# 记忆与混合检索策略

## 短期记忆

短期记忆适合维护最近几轮对话。常见方案是滑动窗口加摘要压缩：窗口保留近几轮原文，摘要保留之前阶段的重要结论。

## 长期记忆

长期记忆适合存储稳定事实，例如用户身份、偏好、项目背景和常用约束。长期记忆通常需要向量化存储，支持按语义相似度召回。

## 混合检索

只做 BM25 容易漏掉语义近似，只做向量检索又可能丢失精确关键词。生产环境通常会把 BM25 和向量检索并行执行，再用 RRF 做融合排序。

## Context Inspector

把摘要、事实与检索来源展示出来，有助于观察模型到底依据了哪些上下文，也方便定位检索错召回和记忆污染问题。
